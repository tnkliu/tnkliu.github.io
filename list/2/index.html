CentOS 7 的一些设置

CentOS 7 有些变化，其中防火墙从iptables变成了firewall，等等逐步整理中。。。

修改ssh端口

vi /etc/ssh/sshd_config

增加端口（避免操作失误，是增加不是修改）

找到
Port 22
把前面的#号去掉
然后复制一行在下面加入（比如增加12222这个端口）
Port 12222

修改firewall

firewall-cmd --zone=public --add-port=12222/tcp --permanent
firewall-cmd --reload
firewall-cmd --zone=public --query-port=12222/tcp
firewall-cmd --list-all
备注：下面是对局域网开放所有端口，不用在这里运行
firewall-cmd --zone=public --add-source="192.168.1.0/24" --permanent
如果是删除端口那么如下命令
firewall-cmd --zone=public --remove-port=12222/tcp --permanent

修改SELinux

安装semanage命令
yum install policycoreutils-python
semanage port -l | grep ssh
semanage port -a -t ssh_port_t -p tcp 12222
semanage port -l | grep ssh

重启ssh

systemctl restart sshd.service

测试新端口链接，成功后记得注释掉 /etc/ssh/sshd_config 中的 Port 22 ，然后重启下ssh即可，这样就把22换成12222了

修改区域或语言

vi /etc/locale.conf

可对应设置，比如

LANG="en_US.UTF-8"

可以用命令操作

localectl status
localectl set-locale LANG=en_US.utf8
localectl set-keymap cn
localectl set-x11-keymap cn

Docker命令

CentOS安装docker

yum install docker-engine

CentOS启动docker服务

service docker start
chkconfig docker on
或
systemctl start docker.service
systemctl enable docker.service

所有的镜像

docker images -q

正在运行的容器

docker ps -a -q

搜索镜像

docker search centos

下载镜像(CentOS Linux release 7.2.1511 (Core))

docker pull centos

启动centos容器并启用bash终端

docker run -i -t centos /bin/bash

接着开始的命令，如果有操作并有更新同时想保存更新，那么需要记住操作终端ID（42ed5222bf49，不是镜像ID，类似[root@42ed5222bf49 /]），然后commit一下(创建一个新的镜像，默认镜像是不会被修改的)

docker commit -m "add vim wget" 42ed5222bf49 mycentos

导出镜像(和docker export类似，export不保留提交历史，无法回滚)

docker save -o mycentos.tar mycentos
或
docker save mycentos > mycentos.tar

导入镜像(和docker import类似，import不保留提交历史，无法回滚)

docker load < mycentos.tar

端口映射和挂载本地目录(ro只读权限)(mytest是一个别名)

docker run -i -t --name mytest -p 80:80 -v /test:/test:ro mycentos /bin/bash

启动关闭的容器

docker start mytest

重新进入容器

docker attach mytest

进入容器

docker exec -it mytest bash

查看包括容器IP等详细信息

docker inspect mytest

停止所有的容器

docker stop $(docker ps -a -q)
或
docker stop (docker ps -a -q)

根据Dockerfile编译

docker build -t nginx-php-fpm .

快捷的zsh和fish shell

把终端升级了一下，经常操作的平台，唯快不破，索性分析了一下zsh和fish两个shell工具，两个都介绍一下。

终端，你需要通过它操作系统，记得从Windows切换到Linux的时候，对命令行又是神秘又是敬畏，怕不会操作，怕有误操作，但是熟悉后就能自主的控制文件，运行命令，可以自定义很多功能，慢慢熟悉后会非常喜欢，会越来越少的误操作，越来越知道什么是命令之美，需要多操作，多使用。

  • zsh shell
  1. yum命令可以直接安装
    yum install zsh
  2. 安装好以后执行
    chsh -s /bin/zsh
  3. 安装 oh-my-zsh 设置zsh主题
    sh -c “$(curl -fsSL https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh)”
  4. 安装 autojump 以便实现 j xxx 快速跳转到目录
    yum install autojump-zsh
  • fish shell
  1. 先到官网下载安装包 http://fishshell.com
  2. 安装好以后执行
    chsh -s /usr/local/bin/fish
  3. 重启终端
  4. 运行 fish_config 会打开一个网页，可以在浏览器中设置终端配色等
  5. 安装 oh-my-fish 可以通过 https://github.com/oh-my-fish/oh-my-fish 查看具体详情
    curl -L https://github.com/oh-my-fish/oh-my-fish/raw/master/bin/install > install
    fish install
  6. 切换主题，输入 omf theme 可以查看所有的主题，然后直接后面接对应的主题回车即可，比如 omf theme lambda，如果提示需要安装，直接omf install对应的主题即可
  7. 如果有自定义环境变量，需要按照如下方式定义
    set -x PATH $PATH /my/bin
  8. 好好享受fish shell带来的快捷把。

Linux服务器命令

查看某端口的连接数

netstat -nat | grep 6379 | wc -l

查看 TCP 连接情况

netstat -n | awk '/^tcp/ {++S[$NF]} END {for(a in S) print a, S[a]}'

按照内容搜索文件(其中 ./ 是服务器的目录)

grep "搜索关键字" -r ./

按照文件名搜索文件

find ./ -name filename

搜索并删掉

ps -ef | grep 'nginx' | awk '{print $2}' | xargs kill -9

查看系统版本

uname -r
cat /etc/issue
cat /proc/version

查看具体命令放在什么位置

which nginx

修改内核可以同时打开的文件描述符的最大值

ulimit -SHn 65535

查看某一端口占用情况

lsof -i :11211

查看内存条数

dmidecode -t memory | grep Size

修改服务器主机名

hostnamectl set-hostname myserver

设置ssh密钥

ssh-keygen -t rsa -C "your name"

tar 加密

tar -zcvf - xxx | openssl des3 -salt -k 123321 | dd of=xxx.des3

tar 解密

dd if=xxx.des3 | openssl des3 -d -k 123321 | tar zxf -

对文件内容排序(根据每行前面6个字符)

sort -k1.1,1.6 no.txt > ok.txt
