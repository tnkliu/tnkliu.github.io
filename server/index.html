CentOS 下 vsftpd 安装配置

0、特别注意：配置文件中每行末尾不能有多余的空格

1、安装(可编译可yum自行决定，下面db4部分是解决密码认证问题的)

yum install vsftpd
yum install db4 db4-utils

2、配置文件

vim /etc/vsftpd/vsftpd.conf

3、设置用户以及认证

3.1、创建用户和密码(下面文件中奇数行是用户名，偶数行是密码)
vim /etc/vsftpd/vpass.txt

3.2、生成虚拟用户认证db文件

db_load -T -t hash -f /etc/vsftpd/vpass.txt /etc/vsftpd/vpass.db

3.3、编辑认证配置文件

vim /etc/pam.d/vsftpd

3.4、创建虚拟用户权限配置文件(下面/etc/vsftpd/vuser/testuser中的testuser等于vpass中的用户名)

mkdir /etc/vsftpd/vuser
vim /etc/vsftpd/vuser/testuser

4、设置防火墙以及目录权限

4.1、设置Selinux

setsebool -P ftp_home_dir=1
sersebool -P allow_ftpd_full_access=1

4.2、设置目录权限(根目录需要设置成755，主目录设置为777)

mkdir -p /ftp/www
chmod -R 755 /ftp
chmod -R 777 /ftp/www

4.3、防火墙设置(把21号端口添加进入即可)

如果是Centos7，那么执行 firewall-cmd --zone=public --add-service=ftp --permanent && firewall-cmd --zone=public --add-port=21/tcp --permanent 把端口和服务添加进防火墙
vim /etc/sysconfig/iptables

5、重启iptables和vsftpd

如果是Centos7记得重启对应下面的服务
service iptables restart
service vsftpd restart

Git 命令

常用命令

  • 从远程下载
git clone xxx
  • 设置邮箱
git config user.email "xxx@xxx.com"
  • 设置用户名
git config user.name "xxx"
  • 设置提交到远程安全级别（current是当前分支 matching是所有分支）
git config push.default current
  • 把新建的分支初次提交到远程
git push --set-upstream origin xxx
  • 查看仓库信息
git config --list
  • 查看分支
git branch
  • 查看所有的分支包括远程的
git branch -a
  • 新建分支
git branch xxx
  • 切换分支
git checkout xxx
  • 新建同时切换分支
git checkout -b xxx
  • 切换到远程分支
git branch remotes/origin/xxx
  • 合并其他分支到当前分支
git merge xxx
  • 删除分支
git branch -d xxx
  • 删除远程分支
git push origin --delete xxx
  • 如果从远程 clone 下代码后，需要在远程某一个分支进行开发，可以如下操作
git clone xxx
git checkout -b xxx
git branch --set-upstream-to=origin/xxx xxx
git pull
git push
  • 把本地分支提交到 Github 上
mkdir xxx
cd xxx
echo "# xxx" >> README.md
git init
git add README.md
git commit -m "first commit"
git remote add origin https://github.com/xxx/xxx.git
git push -u origin master
  • 本地创建 Git 仓库
git init --bare xxx.git
  • 添加远程仓库
git remote add origin https://xxx.git
  • 取消对文件的修改
git checkout -- <xxxfile>
  • 取消上一次的add操作
git reset HEAD
  • 本地仓库文件误删恢复
git log // 找到提交的 commitid
git reset --hard commitid
  • git常用操作命令
git log 查看log
git reset --hard xxxxxxxxxxxxxxxxxxxx 恢复版本到指定log
git pull 提交到远程
git status 查看状态
git checkout -b new_xxx 创建分支并切换到分支
git push --set-upstream origin new_xxx 提交新建的本地分支到远程
git branch -d xxx 删除分支
git branch -D xxx 强制删除分支
git push origin --delete xxx 删除远程分支

配置Rsync服务

安装rsync或者使用系统默认的rsync，然后做如下的配置：

vim /etc/rsyncd.conf

pid file = /etc/rsyncd/rsyncd.pid
port = 873
address = 192.168.1.2
uid = root
gid = root
use chroot = yes
read only = no
hosts allow = 192.168.1.0/24
hosts deny = *
max connections = 5
motd file = /etc/rsyncd/rsyncd.motd
log file = /etc/rsyncd/rsyncd.log
log format = %t %a %m %f %b
timeout = 300
secrets file = /etc/rsyncd/rsyncd.secrets
[www]
path = /www
comment = rsync files
auth users = root
list = true
ignore errors
secrets file = /etc/rsyncd/rsyncd.secrets
[api]
path = /api
comment = rsync files
auth users = root
list = true
ignore errors
secrets file = /etc/rsyncd/rsyncd.secrets

如果没有 /etc/rsyncd/ 目录，可以自行创建。
然后新建并且添加如下的内容：

vim /etc/rsyncd/rsyncd.motd

Hello World

vim /etc/rsyncd/rsyncd.password

123456

vim /etc/rsyncd/rsyncd.secrets

root:123456

编辑好文件后，这几个文件权限都变成 600 权限，不然会遇到很多坑。
然后把上面的文件可以复制成多份放到不同的服务器上，只需要修改 /etc/rsyncd.conf 中的address为对应的服务器IP即可。

启动服务

/usr/bin/rsync --daemon --config=/etc/rsyncd.conf

执行同步

/usr/bin/rsync -vzrtopg --delete --exclude="log" --progress  /www/ root@192.168.1.3::www  --password-file=/etc/rsyncd/rsyncd.password >>/etc/rsyncd/rsyncd.log 2>&1

就可以把IP为 192.168.1.2 的 www 下的内容同步到 192.168.1.3 的 www 节点了。
有个点需要注意下，就是上面设置的端口 873 ，记得需要对防火墙授权。
然后密码可以随意设置，只需要保证 password 和 secrets 文件中的密码一样即可。

CentOS 7 的一些设置

CentOS 7 有些变化，其中防火墙从iptables变成了firewall，等等逐步整理中。。。

修改ssh端口

vi /etc/ssh/sshd_config

增加端口（避免操作失误，是增加不是修改）

找到
Port 22
把前面的#号去掉
然后复制一行在下面加入（比如增加12222这个端口）
Port 12222

修改firewall

firewall-cmd --zone=public --add-port=12222/tcp --permanent
firewall-cmd --reload
firewall-cmd --zone=public --query-port=12222/tcp
firewall-cmd --list-all
备注：下面是对局域网开放所有端口，不用在这里运行
firewall-cmd --zone=public --add-source="192.168.1.0/24" --permanent
如果是删除端口那么如下命令
firewall-cmd --zone=public --remove-port=12222/tcp --permanent

修改SELinux

安装semanage命令
yum install policycoreutils-python
semanage port -l | grep ssh
semanage port -a -t ssh_port_t -p tcp 12222
semanage port -l | grep ssh

重启ssh

systemctl restart sshd.service

测试新端口链接，成功后记得注释掉 /etc/ssh/sshd_config 中的 Port 22 ，然后重启下ssh即可，这样就把22换成12222了

修改区域或语言

vi /etc/locale.conf

可对应设置，比如

LANG="en_US.UTF-8"

可以用命令操作

localectl status
localectl set-locale LANG=en_US.utf8
localectl set-keymap cn
localectl set-x11-keymap cn
